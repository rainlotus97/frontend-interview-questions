## 1. GET和POST有什么区别？

GET和POST是HTTP协议中的两种请求方法，主要区别：

| 区别 | GET | POST |
|------|-----|------|
| 请求参数位置 | 参数附加在URL之后 | 参数包含在请求体中 |
| 请求长度限制 | 受浏览器URL长度限制 | 没有限制 |
| 安全性 | 参数暴露在URL中，不能传输敏感信息 | 参数在请求体中，相对更安全 |
| 幂等性 | 幂等，多次执行返回相同结果 | 非幂等，每次提交创建新资源 |
| 缓存 | 可以被缓存 | 不会缓存（除非特殊设置） |
| 后退/刷新 | 可以通过后退按钮重复执行 | 不会 |

## 2. HTTP2相对于HTTP1.x有什么优势和特点？

HTTP/2相对于HTTP/1.x具有显著的优势：

1. **二进制分帧层**：将所有传输的信息分割为更小的消息和帧（frame），并以二进制格式进行编码
2. **多路复用**：允许在单个TCP连接中并行处理多个请求和响应，消除了HTTP/1.x中的队头阻塞问题
3. **头部压缩**：通过共享头部信息，显著减少传输的数据量
4. **服务器推送**：允许服务器主动向客户端推送资源，减少往返时间
5. **流量控制**：通过流控制、消息控制和窗口控制等机制，实现了对流量的精细控制

## 3. https是怎么保证安全的，为什么比http安全

HTTPS比HTTP更安全主要通过以下机制：

1. **加密传输**：使用SSL/TLS协议对HTTP报文进行加密，结合了对称加密和非对称加密
2. **身份验证**：通过数字证书进行身份验证，确保通信双方的真实性
3. **数据完整性保护**：使用消息摘要（hash）算法生成摘要值，确保数据在传输过程中没有被篡改

## 4. HTTP的状态码有哪些？

| 状态码 | 类型 | 含义 |
|--------|------|------|
| 1xx | 信息性 | 100 Continue |
| 2xx | 成功 | 200 OK, 201 Created, 204 No Content |
| 3xx | 重定向 | 301 Moved Permanently, 302 Found, 304 Not Modified |
| 4xx | 客户端错误 | 400 Bad Request, 401 Unauthorized, 403 Forbidden, 404 Not Found |
| 5xx | 服务器错误 | 500 Internal Server Error, 501 Not Implemented, 503 Service Unavailable |

## 5. post请求为什么会多发送一次option请求

`POST` 请求前发送的 `OPTIONS` 请求实际上是 HTTP 的"预检请求"（Preflight request）。这主要发生在跨域请求（CORS）的场景中。

预检请求的目的是检查服务器是否允许来自不同源（域、协议或端口）的请求进行某些操作。

## 6. http的请求和响应报文分别是什么样的？

**HTTP请求报文**：请求行 + 请求头部 + 空行 + 请求正文

**HTTP响应报文**：状态行 + 响应头部 + 响应正文

## 7. 同样是重定向307，303，302的区别？

| 状态码 | 临时性 | 请求方法改变 | 协议版本 |
|--------|--------|-------------|----------|
| 302 | 临时重定向 | 允许POST变GET | HTTP 1.0 |
| 303 | 临时重定向 | 只允许变为GET | HTTP 1.1 |
| 307 | 临时重定向 | 不允许POST变GET | HTTP 1.1 |

## 8. HTTP的keep-alive是干什么的？

HTTP的keep-alive（HTTP长连接）是一种通过重用TCP连接来发送和接收多个HTTP请求的机制。主要作用：

1. 减少连接建立开销
2. 降低网络负载
3. 提高性能和响应时间
4. 支持HTTP管道化

## 9. 从输入URL到看到页面发生的全过程

1. **用户输入URL**：在浏览器地址栏输入URL
2. **浏览器解析URL**：检查本地hosts文件或通过DNS解析获取IP地址
3. **建立TCP连接**：进行三次握手
4. **发送HTTP请求**：向服务器发送请求
5. **服务器处理请求**：查询数据库、生成动态内容等
6. **发送HTTP响应**：返回响应给浏览器
7. **浏览器解析和渲染页面**：解析HTML、下载CSS/JS资源、渲染页面

## 10. 浏览器缓存的优先级？

浏览器缓存的优先级顺序：

1. **Service Worker 缓存**：具有最高的优先级
2. **Memory Cache（内存缓存）**：当前页面中已经抓取到的资源
3. **HTTP Cache**：
   - 强制缓存：Expires 和 Cache-Control
   - 协商缓存：Last-Modified/If-Modified-Since 和 Etag/If-None-Match
4. **Disk Cache（磁盘缓存）**：存储在硬盘中的缓存
5. **Push Cache（推送缓存）**：HTTP/2中的内容

## 11. 为什么会存在跨域及常见跨域的解决办法？

跨域问题的存在主要是因为浏览器的同源策略限制。

**常见解决方案：**

1. **JSONP**：利用`<script>`标签不受同源策略限制的特性
2. **CORS（跨域资源共享）**：通过服务器端设置相应的HTTP头部信息
3. **代理服务器**：通过搭建代理服务器来转发请求
4. **window.postMessage**：HTML5引入的新API
5. **设置document.domain**：适用于同一顶级域名下的不同子域名

## 12. 浏览器的渲染机制是怎样的？

1. **解析HTML并构建DOM树**
2. **解析CSS并构建CSSOM树**
3. **构建渲染树**（DOM树 + CSSOM树）
4. **布局**（计算每个节点的位置和大小）
5. **绘制**（将节点转换为像素）

## 13. 什么是重绘和回流及怎么减少重绘和回流？

**重绘**：当页面中元素样式的改变不影响布局时触发。

**回流**：当页面布局或几何属性发生变化时触发，浏览器需要重新计算元素的几何属性。

**减少重绘和回流的方法：**

1. 避免频繁操作样式
2. 利用CSS3动画（GPU加速）
3. 避免使用table布局
4. 批量修改DOM（DocumentFragment）
5. 使用绝对定位
6. 避免使用内联样式

## 14. 浏览器的事件机制

事件的传播过程分为三个阶段：**捕获阶段**、**目标阶段**、**冒泡阶段**。

## 15. 浏览器的事件循环和node的事件循环区别

- 浏览器中的事件循环运行在Javascript引擎和渲染引擎之间
- Node.js中的事件循环运行在单独的线程中，基于事件驱动模型
- 在宏任务和微任务的处理上，两者也有所不同

## 16. 浏览器的垃圾回收机制

JavaScript具有自动垃圾回收机制，主要通过两种实现方式：**标记清除**和**引用计数**。

## 17. 进程和线程的区别?

| 区别 | 进程 | 线程 |
|------|------|------|
| 资源占用 | 有独立的地址空间和系统资源 | 共享进程的资源 |
| 并发性 | 切换开销大 | 切换开销小 |
| 独立性 | 独立运行 | 依赖进程 |
| 系统开销 | 创建和撤销开销大 | 创建和撤销开销小 |

## 22. tcp和udp的区别

| 区别 | TCP | UDP |
|------|-----|-----|
| 连接性 | 面向连接 | 无连接 |
| 可靠性 | 提供可靠传输 | 不保证可靠传输 |
| 效率 | 较低 | 较高 |
| 流量控制 | 有 | 无 |
| 应用场景 | 文件传输、电子邮件 | 视频直播、在线游戏 |

## 23. 简述tcp三次握手和4次挥手的过程

**三次握手：**
1. 客户端发送SYN包到服务器
2. 服务器发送SYN+ACK包到客户端
3. 客户端发送ACK包到服务器

**四次挥手：**
1. 客户端发送FIN包到服务器
2. 服务器发送ACK包到客户端
3. 服务器发送FIN包到客户端
4. 客户端发送ACK包到服务器

## 24. tcp/ip协议包含哪几层

1. **网络接口层**：处理与电缆的物理接口细节
2. **网络层**：处理分组在网络中的活动（IP、ICMP、IGMP、ARP）
3. **传输层**：提供端到端的通信（TCP、UDP）
4. **应用层**：处理特定的应用程序细节
