## 1.GET和POST有什么区别？

GET和POST是HTTP协议中的两种请求方法，它们之间存在一些重要的区别：

1. **请求参数的位置**：GET请求的请求参数会附加在URL之后，而POST请求的请求参数则包含在请求体中。
2. **请求长度的限制**：GET请求的长度受限于浏览器对URL长度的限制，而POST请求则没有这个问题。
3. **安全性**：GET请求的参数会暴露在URL中，因此不能用于传输敏感信息。而POST请求的参数在请求体中，相对更加安全。
4. **幂等性**：GET请求是幂等的，而POST请求则不是幂等的。
5. **缓存**：GET请求可以被缓存，而POST请求则不会。
6. **后退/刷新按钮的影响**：GET请求可以被浏览器缓存，因此可以通过点击后退按钮或刷新按钮来重复执行。

## 2.HTTP2相对于HTTP1.x有什么优势和特点？

1. **二进制分帧层**：HTTP/2不再使用文本格式来传输数据，而是将所有传输的信息分割为更小的消息和帧（frame），并以二进制格式进行编码。
2. **多路复用**：HTTP/2引入了多路复用技术，允许在单个TCP连接中并行处理多个请求和响应。
3. **头部压缩**：HTTP/2使用了头部压缩技术，通过共享头部信息，可以显著减少传输的数据量。
4. **服务器推送**：HTTP/2允许服务器主动向客户端推送资源，而无需等待客户端的请求。
5. **流量控制**：HTTP/2通过流控制、消息控制和窗口控制等机制，实现了对流量的精细控制。

---

## 3.https是怎么保证安全的，为什么比http安全

HTTPS之所以比HTTP更安全，主要是通过以下几种机制：

1. **加密传输**：HTTPS使用SSL/TLS协议对HTTP报文进行加密，使得敏感数据在网络传输过程中不容易被窃听和篡改。
2. **身份验证**：HTTPS通过数字证书进行身份验证，确保通信双方的真实性。
3. **数据完整性保护**：在传输数据之前，HTTPS会对数据进行加密，并使用消息摘要（hash）算法生成一个摘要值。

## 4.HTTP的状态码有哪些？

**1xx（信息性状态码）：**
- **100 Continue**：客户端应继续其请求。

**2xx（成功状态码）：**
- **200 OK**：请求成功。
- **201 Created**：请求已经被实现，并因此创建了一个新的资源。
- **204 No Content**：服务器成功处理了请求，但没有返回任何内容。

**3xx（重定向状态码）：**
- **301 Moved Permanently**：请求的资源已被永久移动到新的URL上。
- **302 Found**：请求的资源现在临时从不同的URL响应请求。
- **304 Not Modified**：客户端已经执行了GET请求，但文件未发生变化。

**4xx（客户端错误状态码）：**
- **400 Bad Request**：服务器无法理解请求。
- **401 Unauthorized**：请求要求进行身份验证。
- **403 Forbidden**：服务器理解请求，但拒绝执行它。
- **404 Not Found**：服务器无法找到请求的资源。
- **405 Method Not Allowed**：请求中指定的方法不被允许。

**5xx（服务器错误状态码）：**
- **500 Internal Server Error**：服务器遇到了一个未曾预料的情况。
- **501 Not Implemented**：服务器不支持当前请求所需要的某个功能。
- **503 Service Unavailable**：由于临时的服务器维护或者过载，服务器当前无法处理请求。

## 5.post请求为什么会多发送一次option请求

`POST` 请求前发送的 `OPTIONS` 请求实际上是 HTTP 的一种特性，称为"预检请求"（Preflight request）。这主要发生在跨域请求（CORS, Cross-Origin Resource Sharing）的场景中。

预检请求的目的是检查服务器是否允许来自不同源（域、协议或端口）的请求进行某些操作。

## 6.http的请求和响应报文分别是什么样的？

HTTP请求报文主要由请求行、请求头部、空行和请求正文组成。

HTTP响应报文则主要由状态行、响应头部和响应正文组成。

## 7.同样是重定向307，303，302的区别？

1. **临时性与永久性**：307和302都是临时重定向，而303虽然也属于临时重定向，但其语义稍有不同。
2. **请求方法的改变**：
   - 302允许各种各样的重定向，但并不能确保POST请求会重定向为POST。
   - 303只允许任意请求重定向到GET。
   - 307与302类似，但不允许POST请求被重定向为GET请求，即它会保持原有的请求方法不变。
3. **协议版本**：302和301是HTTP 1.0中的内容，而303、307等则是HTTP 1.1中引入的。

## 8.HTTP的keep-alive是干什么的？

HTTP的keep-alive，也称为HTTP长连接，是一种通过重用TCP连接来发送和接收多个HTTP请求的机制。

1. **减少连接建立开销**
2. **降低网络负载**
3. **提高性能和响应时间**
4. **支持HTTP管道化**

## 9.从输入URL到看到页面发生的全过程

1. **用户输入URL**：用户首先在浏览器地址栏中输入想要访问的网站的URL。
2. **浏览器解析URL**：浏览器内部代码会解析这个URL，检查本地hosts文件，或将域名发送给DNS服务器进行解析。
3. **建立TCP连接**：浏览器得到IP地址后，会通过TCP协议与服务器建立连接（三次握手）。
4. **发送HTTP请求**：TCP连接建立后，浏览器会向服务器发送HTTP请求。
5. **服务器处理请求**：服务器收到请求后，根据请求的内容进行相应的处理。
6. **发送HTTP响应**：服务器处理完请求后，会发送一个HTTP响应给浏览器。
7. **浏览器解析和渲染页面**：浏览器收到响应后，解析HTML代码，下载所需的CSS、JavaScript等资源文件，然后渲染页面。

## 11.为什么会存在跨域及常见跨域的解决办法？

## 13.什么是重绘和回流及怎么减少重绘和回流？

**重绘**（Repaint）：当页面中元素样式的改变不影响布局时，浏览器将只会重新绘制受影响的元素。

**回流**（Reflow）：当页面布局或几何属性发生变化时，浏览器需要重新计算元素的几何属性。

**减少重绘和回流的方法：**

1. 避免频繁操作样式
2. 利用CSS3动画
3. 避免使用table布局
4. 批量修改DOM
5. 使用绝对定位
6. 避免使用内联样式
7. 优化图片加载
8. 利用浏览器的缓存机制

## 17.进程和线程的区别?

1. **资源占用**：进程是系统分配资源的基本单位，线程是进程的一个实体。
2. **并发性**：线程间的切换开销小，进程间的切换开销较大。
3. **独立性**：进程是独立运行的，线程共享进程的资源。
4. **系统开销**：创建或撤销进程的开销远大于创建或撤销线程的开销。

## 19.web安全攻击方式及防御方法

**1、SQL注入** - 使用参数化查询、输入验证

**2、跨站脚本攻击 (XSS)** - 对用户输入进行验证和过滤、设置CSP

**3、跨站请求伪造 (CSRF)** - 使用CSRF令牌、验证referer头部

**4、未经授权的访问** - 加强身份认证、实施访问控制

**5、重定向攻击** - 审核重定向的网址，通过加入黑白名单的方式监控

**6、文件上传漏洞** - 严格限制上传文件类型和大小

**7、目录遍历攻击** - 限制用户访问权限、过滤用户输入

**8、代码执行漏洞** - 将应用程序运行在沙盒环境中

**9、点击劫持攻击** - 使用X-Frame-Options头部

**10、cookie攻击** - 使用HttpOnly属性

**11、DOS攻击** - 通过防火墙策略和报警机制来解决

## 22.tcp和udp的区别

1. **连接与无连接**：TCP是面向连接的协议，UDP是无连接的协议。
2. **可靠性**：TCP提供了可靠的数据传输服务，UDP不保证数据包的可靠传输。
3. **效率**：UDP比TCP更高效。
4. **流量控制**：TCP具有流量控制功能，UDP没有。
5. **应用场景**：TCP通常用于需要可靠传输的场景，UDP适用于对实时性要求较高的场景。

## 23.简述tcp三次握手和4次挥手的过程

**三次握手过程**：

1. 客户端发送SYN包到服务器
2. 服务器发送SYN+ACK包到客户端
3. 客户端发送ACK包到服务器

**四次挥手过程**：

1. 客户端发送FIN包到服务器
2. 服务器发送ACK包到客户端
3. 服务器发送FIN包到客户端
4. 客户端发送ACK包到服务器

---

## 24.tcp/ip协议包含哪几层

TCP/IP协议族是一个网络通信模型，包含了四层结构：

1. **网络接口层**：处理与电缆的物理接口细节
2. **网络层**：处理分组在网络中的活动，包括IP、ICMP、IGMP以及ARP
3. **传输层**：主要为两台主机上的应用程序提供端到端的通信，包括TCP和UDP
4. **应用层**：负责处理特定的应用程序细节
